Importing TPTP Runtime Traces

The TPTP sphere requires loading one or more previously obtained dynamic runtime traces. To load such a trace, use the following steps:

  1. From the Ferret Sphere Configuration wizard, select the Java dynamic run-time queries (TPTP) sphere type:

  2. This will bring up the TPTP Profile Selection wizard:

    Select the TPTP Profile Import Wizard link.

  3. This wizard aids in loading a runtime trace (or profile). Use the Browse... button to find your trace file in the file system. Then be sure to select the Show full data (execution flow graphical details) radio button.

    Click Finish to begin the import. This will return to the TPTP Profile Selection wizard

  4. The newly-loaded trace should now be visible in the wizard, and should have been automatically selected.

  5. You may wish to set the fidelity of the selected trace using the provided combo box.

  6. Select Finish to complete the configuration of this TPTP sphere instance.

Troubleshooting

Should you be receiving errors when attempting to import a profile, please ensure that:

  1. You are importing the profile with the Show full data (execution flow graphical details) option selected, as described above. To remove previously loaded profiles, open the Profiling Monitor view, select the appropriate profiles, and select Delete from the context menu.

  2. The trace was created with the Show execution flow graphical details.
